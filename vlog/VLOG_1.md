| **0:00–0:40** | **Hook**         | "I thought I was designing a website. I ended up designing a process. Because the website isn't the product—it's just the first public face of Flying Fortress Games. If we got that wrong, we'd spend years slowly correcting decisions that never should have been made in the first place."                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| **0:40–1:30** | **Context**      | "Flying Fortress Games doesn't have a released game yet. No marketing department. No design team. Just an idea of the kind of studio we want to become. Before choosing colors or fonts, we tried answering a more difficult question: who are we, really? What values do we want people to feel before they've read a single line of text?"                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| **1:30–2:45** | **The Problem**  | "The game industry already has a visual language. Dark interfaces. Neon accents. Minimal dashboards. They're good designs, and they work well for many studios. The problem wasn't that they looked bad. The problem was that they didn't feel like *us*. We weren't looking for a beautiful website. We were looking for a visual language that honestly reflected Flying Fortress Games."                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| **2:45–5:00** | **The Process**  | "So we approached the problem like engineers. First, define the objective. Then explore. Compare. Build a vocabulary. Eliminate ambiguity. Write specifications. Only then start implementing.<br><br>We explored seven directions—and probably looked at three or four times that many.<br><br>The first surprise was that every direction looked convincing.<br><br>Some of them—honestly—looked awesome.<br><br>That was bad. It made the decision much harder than I expected.<br><br>Was looking good the goal? Looking like *Flying Fortress Games* was.<br><br>Once that became the question, the process became much clearer. Each direction expressed different values, different assumptions, even different personalities. The goal wasn't to find the prettiest design. It was to identify the one that best communicated who we wanted to be." |
| **5:00–6:15** | **The Decision** | "The direction we ultimately chose was French Industrial Art Deco. Not because it felt nostalgic, but because it naturally combined engineering, craftsmanship and culture. Every decision—from the color palette to the typography and materials—could be justified by the same underlying idea. Once the identity was clear, the visual choices almost started making themselves."                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| **6:15–7:30** | **Reflection**   | "Looking back, I don't think this process created our identity. It revealed it. The values were already there—we simply needed a systematic way to translate them into design decisions. Just as importantly, documenting every step means those decisions won't quietly drift over time. Future changes now have a reference point instead of relying on memory or personal taste."                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| **7:30–8:00** | **Closing**      | "The website will be one of the first public artifacts of Flying Fortress Games, but it's only the beginning. Every step of this work is documented publicly, not because it's the only way to build a website, but because I want this channel to be an honest engineering journal. Next time, we'll apply the same mindset to another problem—and see whether the process holds."                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
